# procgen-rogue

Маленький ASCII-рогалик на Python с [tcod]. Он намеренно небольшой, но собран
как **основа для развития**: чистые слои, сущности на компонентах, все настройки
в одном месте, а игровая логика работает без экрана — поэтому её можно тестировать
headless (без окна).

```
     #####.######
     #..........#
     #..........#
     #.,.g.~~.O.#      @  ты        #  стена    O  портал     T  торговец
     #..[]...@..#      g  монстр    .  пол      >  лестница   ,  мусор
     #....>..~~..#      %  останки   /  добыча   ~  вода       []  ящик
```

## Что уже есть

- **Только ASCII**, отрисовка сеткой в стиле Dwarf Fortress (окно tcod/SDL).
- **Один герой** (`@`), за которым **следует камера**, прокручивая карту.
- **Процедурные пещеры**: карта по умолчанию строится из многослойного шума
  (генератор «Noise Lab») — клетка становится стеной там, где яркость шума выше
  `0.4`, иначе это пол. (Классический генератор комнат-и-коридоров тоже есть и
  используется в тестах.)
- **Порталы** (`O`): шум дробит пол на несвязные области, поэтому достаточно
  крупные области выстраиваются в случайную **одностороннюю цепочку** — в каждой
  области есть портал, переносящий в *следующую* область, далеко от её собственного
  выхода, так что её нужно пройти. В последней области цепочки портала **нет** —
  там лестница вниз.
- **Спуск** (`>`): в конце цепочки порталов **зайди на лестницу**, чтобы
  провалиться на заново сгенерированный, более глубокий уровень, где монстры
  **сильнее и их больше**. ХП, золото, инвентарь и экипировка переносятся.
- **Торговец** (`T`, в окружении ящиков): врежься в него, чтобы открыть магазин,
  где можно **продавать предметы** (по `100 × уровень` золота) и **покупать
  постоянные улучшения** любой из «предметных» характеристик. Каждое следующее
  улучшение той же характеристики дороже. Торговец есть не на каждом уровне.
- **Декорации**: косметические объекты на полу, пока не мешающие движению и
  обзору — рассыпанный **мусор** (`,`), **лужи воды** (`~`, большими кластерами)
  и **ящики** (`[]`).
- **Процедурные монстры**: у каждого случайные ХП, урон, шансы крита и уворота,
  и **скорость** (0..1 — шанс шагнуть к игроку за ход, поэтому кто-то стоит на
  месте, а кто-то бродит). Суммарная опасность монстра задаёт и его **цвет** (чем
  краснее, тем опаснее), и уровень выпадающей добычи.
- **Радиус атаки + авто-атака**: любой враг в радиусе атаки игрока получает удар
  автоматически каждый ход — если только игрок не тратит ход на *активность*.
  Движение и ожидание активностями не считаются, так что в это время ты дерёшься.
- **Активности**: ходы, которые занимают игрока и подавляют авто-атаку —
  **лечение** (восстановить ХП) и **разведка** (расширить обзор, пока не сдвинешься).
- **Процедурная добыча**: убийство монстра даёт золото и роняет сгенерированный
  **предмет**, уровень которого растёт с опасностью монстра. Имена собираются из
  словарей прилагательных и существительных (выше уровень → больше слов), и у
  каждого предмета несколько **случайных бонусов** (больше на высоких уровнях):
  макс. ХП, урон, радиус атаки, обзор, шанс крита, шанс уворота, сила лечения.
- **Экипировка**: одновременно можно **использовать только два предмета**; их
  бонусы складываются в эффективные характеристики. Остальное лежит в инвентаре.
  На экране инвентаря также виден полный **лист персонажа** (все текущие статы).
- **Туман войны** с **радиусом 10** клеток; исследованные области запоминаются и
  рисуются приглушённо.
- **Главное меню**: Играть / Обучение / Язык / Выход. Esc в игре возвращает в меню,
  **сохраняя** партию (пункт «Играть» становится «Продолжить») до выхода из
  приложения; Esc в обучении сбрасывает его, так что оно всегда начинается заново.
- **Обучение**: маленькая комната 10×10 с текстом-подсказкой на экране и парой
  слабых неподвижных крыс, на которых можно потренировать бой, сбор добычи и
  экипировку.
- **Локализация**: все строки интерфейса/лога лежат в `rogue/lang/en.py` и
  `rogue/lang/ru.py`; язык переключается прямо из главного меню (кнопка «Язык»).
- **Масштабирование окна**: игра пересобирает консоль под текущий размер окна —
  можно тянуть за край, и карта показывает больше. Строки статуса и подсказок сами
  переносятся на несколько рядов, если текст не влезает по ширине.

## Запуск в консоли/окне

Важно: игра использует SDL-бэкенд tcod и открывает **собственное графическое
окно** — она **не** рисует внутри текстового терминала (TTY). Терминал нужен лишь
чтобы ввести команду запуска; графика откроется в отдельном окне. Значит на Linux
требуется графический сеанс (X11/Wayland); в «голой» консоли без дисплея (по SSH
без X-форвардинга, в чистом TTY) окно не откроется.

Нужен Python 3.11+.

### Linux

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
python main.py                 # случайное подземелье
python main.py --seed 42       # воспроизводимое подземелье
```

Если увидишь ошибку вида «No available video device» — значит нет графического
дисплея (например, чистый SSH/TTY). Запускай в графическом окружении или пробрось
X11 (`ssh -X`).

### Windows

PowerShell или cmd:

```powershell
py -m venv .venv
.\.venv\Scripts\activate
pip install -r requirements.txt
py main.py
```

### Управление

| Клавиши | Действие |
|---------|----------|
| Стрелки · numpad · `Q W E / A D / Z X C` | Движение / удар при упоре (8 направлений) |
| `s` · `.` · numpad `5` | Ждать ход |
| `r` | Лечение (активность: восстановить ХП) |
| `f` | Разведка (активность: расширить обзор, пока не сдвинешься) |
| зайти на `>` | Спуститься на уровень ниже |
| врезаться в `T` | Открыть магазин торговца |
| `i` | Открыть / закрыть инвентарь |
| ↑/↓ · `w`/`x` | (в инвентаре / магазине / меню) двигать выбор |
| `Enter` | (инвентарь) надеть/снять · (магазин) купить/продать · (меню) выбрать |
| `Esc` | Закрыть окно / вернуться в главное меню / выйти |

## Тесты

Вся игровая логика не зависит от экрана, так что тесты запускаются где угодно:

```bash
pip install pytest
pytest -q
```

## Архитектура

Код разбит так, чтобы **новые идеи вставлялись без переписывания**. Слои движка и
мира не импортируют рендерер.

```
main.py                 точка входа (разбор аргументов) -> rogue.app.run
rogue/
├── config.py           все настройки в одном dataclass
├── rng.py              сидируемая централизованная случайность
├── geometry.py         направления + помощники расстояний
├── entity.py           Entity = позиция + глиф + набор компонентов
├── components.py       Fighter / MonsterAI / Inventory / Equipment / Loot / …
├── bonuses.py          enum BonusType, общий для предметов и статов
├── items.py            процедурный предмет + генерация имени/бонусов
├── spawn.py            фабрики сущностей (игрок, монстры, торговец, декор)
├── actions.py          глаголы: Move / Melee / Bump / Wait / Heal / Scout
├── input.py            события клавиш -> действия / команды UI
├── engine.py           владеет миром, ведёт ходы, FOV, исход боя, магазин
├── app.py              окно tcod + главный цикл (меню/игра/обучение)
├── lang/               локализация (en.py, ru.py) — все строки интерфейса
├── world/
│   ├── tiles.py        записи тайлов (проходимость/прозрачность + глифы)
│   ├── game_map.py     сетка местности, видимость, сущности, FOV
│   ├── noise.py        генератор поля Noise Lab (только stdlib)
│   ├── procgen.py      генераторы: пещеры из шума (по умолчанию) + комнаты
│   └── tutorial.py     маленькая комната обучения
└── ui/
    ├── camera.py       вьюпорт, следящий за игроком и прижатый к краям
    └── renderer.py     рисует карту, туман, сущности, HUD, лог, оверлеи
```

**Модель ходов:** пошаговая. Игрок выполняет один `Action`; если он тратит ход,
затем ходит каждый монстр, после чего пересчитывается FOV.

## Как расширять (это заложено)

Несколько конкретных «следующих идей» и где их делать:

- **Новый тип монстра** — фабрика в `spawn.py`; другие статы `Fighter` или новый
  AI-компонент. Умнее поведение (бегство, дальний бой) — это новый `take_turn` в
  `components.py`; цикл ходов не трогается.
- **Новый вид бонуса** — добавь `BonusType` в `bonuses.py`, бросок в `items.py` и
  читай его там, где применяется стат (`Fighter` / `engine`). Генерация, инвентарь
  и UI экипировки подхватят автоматически.
- **Новая тема/более редкие имена предметов** — расширь словари или шаблон имени в
  `items.py`; остальное не меняется.
- **Новый глагол игрока** (бросок, зелье, дверь) — подкласс `Action` плюс биндинг в
  `input.py`. Верни `is_activity=True` из его `ActionResult`, чтобы он стал
  *активностью* (занимает игрока, подавляет авто-атаку) — так же, как `HealAction`
  и `ScoutAction`.
- **Новая местность** (вода, лава, двери) — один вызов `new_tile(...)` в
  `tiles.py`; FOV и рендер подхватят сами.
- **Другие стили карт** (BSP, вольты, иная конфигурация шума) — напиши
  `generator(cfg, rng, depth, player) -> (GameMap, player)` и передай в
  `Engine(generator=…)`; всё ниже по потоку не волнует, как построена карта.
- **Новый язык** — добавь файл в `rogue/lang/` с теми же ключами, что в `en.py`, и
  впиши код языка в `config.LANGUAGES`.
